## 一、本轮目标

本轮重点是把 V0.4.4 的后台输出位置、前台多图队列、目录校验和结果访问链路收口，避免继续出现：

- 成品图落入错误嵌套目录。
- 前台只能处理单张图片。
- 用户无法指定输出目录。
- 后端返回字段不稳定导致前台无法显示结果。
- 单张失败影响后续图片。

本轮没有修改 XHR 上传方式，没有改为 `fetch`，也没有破坏 SSE 日志流。

## 四、后端输出目录能力

### 1. `/api/upload`

新增兼容字段：

```text
output_dir
output_profile
output_format
```

仍保留旧字段：

```text

mode

format
```

后端现在会在任务中返回：

```text
output_dir
output_path
output_filename
output_size
final_output_url
preview_output_url
used_custom_output_dir
output_dir_source
output_dir_created
output_path_exists
```

### 2. `/api/output/validate`

功能：

- 检查输出目录是否合法。
- 不存在时按配置自动创建。
- 检查是否可写。
- 如果路径是文件而不是目录，返回失败。

### 3. `/api/output/open`

功能：

- 打开输出目录。
- 只使用 Windows 安全打开方式。
- 自动校验路径是否存在且是目录。

自动化测试阶段没有实际调用打开窗口，避免干扰本地桌面。

`src/DashboardPage.jsx` 已恢复多图队列能力：

- 支持多选图片。

## 九、当前结论

V0.4.4 的输出位置与多图队列能力已经达到可验收状态：

1. 支持默认输出目录。
2. 支持用户自定义输出目录。
3. 支持目录校验。
4. 支持目录打开接口。
5. 支持多图片选择。
6. 支持多图片拖拽。
7. 支持队列顺序处理。
8. 单张失败不影响后续任务。
9. 每张图都有稳定 `final_output_url`。
10. 输出图可以通过 `/api/file/outputs/{filename}` 访问。

## 十、已知限制

1. Codex 沙箱环境无法直接写入用户桌面目录，因此桌面正式目录只做配置恢复，自动化验证使用工作区内测试目录完成。
2. `/api/output/open` 已实现，但自动测试未实际调用，避免打开系统资源管理器干扰桌面。
3. `main.py` 中仍保留部分历史兼容代码，当前生效的是后定义的 V0.4 Web API 构建函数；本轮未做大范围历史清理，避免影响已跑通链路。
