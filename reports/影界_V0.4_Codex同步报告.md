## 一、当前版本结论

V0.4 已从“前后台通车验证”推进到“1080P 稳定交付基线”。本轮重点不是 UI 重做，也不是 2K / 4K / 8K 或 AI 大模型接入，而是保证后台输出尺寸、输出路径、任务字段、真实性验证和 SSE 链路稳定。

当前验证结论：

- 16:9 低于 1080P 的图片会提升到 `1920 x 1080`。
- 竖图已达到高清基线时不会被默认降采样。
- 超宽图保持比例，不会强制变成 `1920 x 1080`。
- 输出图不是原图复制品，`output_changed=True`，`hash_equal=False`。
- 主流程保留 XHR 上传与 SSE 日志，不改回 fetch。
- 普通用户最终输出以 `final_output_url` 为准。

## 三、最新接口契约

### POST /api/upload

请求字段：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `file` | UploadFile | 必填 | XHR FormData 文件字段，必须叫 `file` |
| `mode` | string | `fidelity` | `fidelity` / `text_safe` / `texture` |
| `scale` | string/int | `2` | 兼容旧字段，V0.4 不再作为主输出口径 |
| `format` | string | `png` | 兼容旧字段 |
| `output_profile` | string | `delivery_1080p` | `delivery_1080p` / `preview_light` / `fidelity_original` |
| `output_format` | string | `auto` | `auto` / `png` / `jpg` / `webp` |

返回字段重点：

- `taskId`
- `streamEndpoint`
- `originalUrl`
- `enhancedUrl`
- `final_output_url`
- `task_result`
- `task_report`

返回示例结构：

## 四、V0.4 输出尺寸策略

| 场景 | 策略 | 是否允许默认降采样 |
| --- | --- | --- |
| 16:9 且低于 1080P | 输出 `1920 x 1080` | 否 |
| 已达到高清基线的竖图 | 保持原尺寸，只做清洁增强 | 否 |
| 超宽图 | 保持比例，长边不足时补到约 2048 | 否 |
| `fidelity_original` | 原尺寸忠实增强 | 否 |
| `preview_light` | 轻量交付，但质量守门不通过时回退高清主图 | 否 |

新增关键字段：

- `target_resolution`
- `resize_policy`
- `was_upscaled`
- `was_downscaled`
- `processing_profile`
- `image_type`
- `degradation_type`
- `encoding_warning`

## 六、路径与缓存策略

当前路径策略：

- 用户正式输出目录：`C:\Users\xyppt\Desktop\雪原Ai增强引擎\输出成品\` 或配置中的输出目录。
- 项目运行缓存目录：`runtime/`。
- 输入缓存默认保留，不自动清理。
- `cleanup_input_cache` 已作为预留钩子存在，默认禁用，避免误删用户素材。

本地沙箱测试中，由于桌面路径不在当前写入权限内，输入缓存自动回退到项目 `runtime/v04_inputs`。实际 Windows 桌面运行时会优先使用用户桌面输入目录。

## 七、SSE 日志链路

保留当前异步 SSE 机制：

- 上传后返回 `taskId` 与 `streamEndpoint`。
- 详情页通过 `EventSource` 连接 `/api/v1/tasks/{task_id}/stream`。
- 后端逐条发送 `restoration.log`。
- 最后发送 `[DONE]`。
- 前端收到完成信号后解锁高清对比页。
