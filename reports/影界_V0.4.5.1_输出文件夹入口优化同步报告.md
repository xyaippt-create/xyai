# 影界 / VisualMasterPro V0.4.5.1 输出文件夹入口优化同步报告

生成时间：2026-06-15  
项目路径：`D:\Codex\04_Visual-Master-Pro`

## 一、验收结论

状态：**候选通过**

本轮只调整 `src/DashboardPage.jsx` 中的输出文件夹控制面板，不改动上传协议、不改动后端接口、不改动 1080P 图像处理逻辑。

## 二、本轮问题归因

原面板存在交互认知问题：

1. 入口不清晰：用户看见裸路径输入框，不知道从哪里开始“修改输出文件夹”。
2. 路径显示不完整：长路径容易被截断，无法核对完整目录。
3. “验证”和“应用”分离：用户不确定哪个动作才会对本次处理生效。
4. 状态反馈不足：默认目录 / 自定义目录没有明显状态高亮。

### 3. 修改入口明确化

新增主入口按钮：

```text
修改输出文件夹
```

点击后：

- 展开路径输入框。
- 自动聚焦输入框。
- 默认填入当前生效目录。

### 4. 验证与应用合并

删除旧的双按钮：

```text
验证输出路径
应用输出路径
```

合并为：

```text
验证并应用
```

点击后流程：

```text
POST /api/output/validate
→ 成功后 POST /api/output/apply
→ 更新当前有效 output_dir
→ Badge 切换为 自定义目录
→ 绿字提示：✓ 已成功应用自定义输出文件夹
```

失败时：

```text
不改变当前生效路径
不跳转
不黑屏
红字提示：❌ 路径不可用，请检查目录是否存在或权限是否允许写入
```

### 5. 恢复默认文件夹

按钮：

```text
恢复默认文件夹
```

点击后：

- 清空自定义目录。
- Badge 回到 `当前状态：默认目录`。
- 绿字提示：`✓ 已恢复默认输出位置`。
- 后续上传不再携带自定义 `output_dir`，后端走默认桌面成品目录。

### 6. 打开输出文件夹

按钮：

```text
打开输出文件夹
```

当前逻辑：

```text
POST /api/output/open
```

目标路径：

```text
当前真正生效的自定义目录
或默认输出目录
```

不会误打开输入框里尚未验证并应用的临时文本。
