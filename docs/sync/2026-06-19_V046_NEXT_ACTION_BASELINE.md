# V0.4.6 下一阶段开发基线

生成日期：2026-06-19

## 1. 下一阶段建议

建议进入：

```text
V0.4.6 RC1前｜画质通透度与真实收益增强专项
```

本阶段不是继续扩展前台，也不是进入 2K/4K。主线应回到画质核心和用户肉眼可感知收益。

## 2. 当前产品定位

```text
影界 VisualMasterPro V0.4.6 是面向中文商业视觉交付的本地1080P图片保真增强与交付风险判断工具。
```

核心价值：

```text
保真增强
中文文字保护
Logo与品牌色保护
人物自然保护
材质与边缘稳定
默认色彩稳定
低质量照片保守修复
最终交付风险判断
本地输出与诊断反馈
```

## 3. 下一阶段优先级

1. 通透度增强
2. 低频干净
3. 暗部灰雾控制
4. 高光与渐变洁净
5. 产品与建筑材质层次
6. 真实照片轻中度增强
7. 文件体积收益比
8. 证据包人工分类口径
9. 前台只做基础状态同步

## 4. 开发边界

下一阶段仍必须遵守：

- 不改 API 字段名；
- 不破坏 XHR 上传；
- 不破坏 EventSource SSE；
- 不破坏 `final_output_url` / `preview_output_url`；
- 不把本地盘符路径作为 `<img src>`；
- 不隐藏 `PASS_WITH_LIMITATION`；
- 不做账号、云同步、素材库、营销页；
- 不新增 2K、4K、8K、印刷级或喷绘级承诺；
- 不宣传 AI 重绘、人脸重塑、一键让所有图片明显变清晰。

## 5. 画质专项建议任务拆分

### T01 通透度与低频干净专项诊断

目标：

- 区分“尺寸放大”与“真实清晰度提升”；
- 找出画面发灰、雾蒙、暗部脏、低频块状、平滑区域污染的主要样本；
- 只生成最小诊断表，不扩建复杂平台。

建议样本：

- 真实低质量人物/活动照片；
- 真实产品照片；
- 真实建筑照片；
- 合成渐变/高光；
- 中文小字保护样本；
- 高质量商业 KV 保护样本。

### T02 轻量通透度候选

目标：

- 改善暗部灰雾；
- 降低低频脏污；
- 控制高光和平滑区域色带；
- 保持默认色彩稳定；
- 不做风格化调色，不提高饱和度。

### T03 体积收益比复核

目标：

- 对 `phase6_size_growth_ratio` 与 `phase6_visible_benefit_score` 做人工分类联动；
- 避免输出体积显著增长但肉眼收益有限；
- 不为了压体积牺牲文字、Logo、Alpha 和材质层次。

### T04 证据包文案与分类展示修正

目标：

- 保留当前重新分类口径；
- 修正 `report.md`、`review_index.html`、`review_notes.md` 的中文乱码显示；
- 不重跑算法，不改图片输出；
- 继续强调“100% 裁切优先，整图对比仅用于观察构图和颜色”。

## 6. 当前不建议做的事项

不建议继续投入：

- 复杂 Dashboard 视觉美化；
- 新增 UI 页面；
- 账号系统；
- 云端同步；
- 素材库；
- 大型质量报告系统；
- 2K/4K/8K 入口；
- 印刷或喷绘级承诺。

## 7. 下一阶段准入条件

开始画质专项前应确认：

- 当前未提交代码改动是否需要先独立收口；
- 证据包分类口径已经作为人工判断标准；
- 下一阶段只选择少量决策样本，不直接全量重跑 19 张；
- 若涉及算法候选，必须通过正式链路：

```text
main.py
→ engine.pipeline.process_v046_delivery
→ v046_1080p_delivery
→ v046_delivery_adapter
→ process_v036_output
→ engine/algorithms/*
→ safe_copy_final
→ final_output_url
```
