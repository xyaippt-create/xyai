# V0.4.6 已知问题

生成日期：2026-06-18

## 1. 允许进入后续阶段但必须保留的问题

- 历史文件体积扩张仍存在，Phase 6 已增加交付层标记，但未承诺完全解决；
- 部分普通 PNG、中文小字、合成渐变和细线样本仍可能 `quality_1080p_pass=false`；
- 部分高质量或合成样本会因保护策略跳过或接近 0 强度处理；
- 华为样本历史色彩漂移归入 Phase 5，不在 Phase 6 修改；
- 真实主动色偏修复样本仍为 `SAMPLE_GAP`；
- Phase 6 的交付状态是工程守门结果，不等同于用户肉眼最终裁判。

## 2. Phase 6 新增限制说明

出现以下情况时，输出可能为 `PASS_WITH_LIMITATION`：

- 历史 1080P 质量门未完全通过；
- 文件体积增长明显但可见收益不足；
- 平滑区域、高光或渐变风险较高；
- 文字或合成图需要人工复核。

## 3. 不得误写为已完成

不得写成已经完成：

- 文件体积全局优化；
- 所有 PNG 和中文小字样本通过质量门；
- 主动色偏修复真实样本闭环；
- 2K、4K、印刷或喷绘级能力；
- 前台 UI 与 Gemini 阶段。

## 4. RC1 前建议

- UI 必须展示 `final_delivery_status` 与 `final_delivery_reason`；
- 对 `PASS_WITH_LIMITATION` 提供人工查看入口；
- 补齐真实主动色偏修复样本；
- 保留反馈 ZIP 导出入口，但不得把用户原图自动打包。
