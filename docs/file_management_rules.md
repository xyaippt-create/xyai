# VisualMasterPro 文件管理规范

## 目标

VisualMasterPro 是长期演进的 AI 商业视觉质量引擎。项目文件必须区分源码、规则、输入素材、正式输出、测试输出和缓存，避免图片、报告、临时文件混在同一层目录。

## 根目录职责

- `main.py`：统一命令行入口，负责接收输入、输出、模式和报告参数。
- `engine/`：核心引擎代码，包括分析、算法、规则解释、图像读写和总编排。
- `pipelines/`：商业视觉模式管线。不同 mode 在这里拥有独立处理流程。
- `modes/`：视觉模式定义和模式注册。
- `rules/`：全局视觉规则库。
- `ai_noise_rules/`：AI 脏感、diffusion 纹理、假 HDR、电子锐边等污染规则。
- `material_rules/`：材质规则，包括玻璃、金属、皮肤、食物、建筑等材质逻辑。
- `visual_style_rules/`：商业视觉风格规则，包括 KV、电影感、PPT 商业、人像、奢侈品等风格倾向。
- `configs/`：默认配置和后续运行配置。
- `docs/`：架构说明、测试报告、文件管理规范和阶段性决策记录。
- `tests/`：测试用例、测试说明和测试辅助资源。
- `scripts/`：历史脚本、兼容入口和打包脚本。稳定历史版本不随意删除。
- `输入图片/`：待处理原图。禁止覆盖原图。
- `输出成品/`：正式输出根目录，只保存运行生成的成品、报告、对比图和归档数据。

## 默认交付输出目录

```text
输出成品/
└── images/
```

- `images/`：最终商业交付图。默认每次运行只输出一张图片。

## 开发调试输出

仅在 `--debug` 或 `--developer` 模式下生成以下目录：

- `reports_json/`：机器可读的质量报告 JSON。
- `reports_md/`：人工阅读的质量报告 Markdown。
- `compare/`：前后对比图、模式对比图。

`quality_report` 默认只作为内部调试数据参与商业信息角标生成，不再作为正式交付文件输出。

## 测试输出

- 临时测试图和测试报告应放入 `tests/outputs/`。
- 不要把临时测试图混入 `输出成品/`。
- 阶段性测试总结继续放入 `docs/`，例如 `docs/test_report_v3.md`。

## Git 忽略规则

以下内容不应进入 Git：

- Python 缓存：`__pycache__/`、`*.pyc`
- 本地虚拟环境：`.venv/`、`venv/`
- 打包缓存：`build/`、`dist/`
- EXE 打包中间文件：`*.spec`
- 测试输出：`tests/outputs/`
- 输出归档：`输出成品/archive/`
- 临时日志：`*.tmp`、`*.log`

## 文件命名

- 正式输出图片命名格式：`原文件名_mode_雪原Ai·PPT设计.png`
- JSON 报告命名格式：`原文件名_mode_雪原Ai·PPT设计_quality_report.json`
- Markdown 报告命名格式：`原文件名_mode_雪原Ai·PPT设计_quality_report.md`

## 禁止事项

- 禁止覆盖 `输入图片/` 内原图。
- 禁止把正式输出、测试输出和缓存混在同一目录。
- 禁止把缓存、打包产物和临时文件提交到 Git。
- 禁止删除未确认的输入素材、历史脚本和规则库。
