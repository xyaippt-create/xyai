# VisualMasterPro 软件版本历史

本文档用于记录 VisualMasterPro（雪原Ai增强引擎）的长期版本演进，服务于 Codex 开发同步、GitHub 项目记录、商业化迭代管理、用户更新日志和后续收费版本规划。

## V1.0：基础图像处理阶段

### 版本定位

V1.0 是 VisualMasterPro 的基础验证版本，目标是完成本地图像读取、基础处理和输出保存，验证 AI 图片增强流程的可执行性。

### 核心升级

- 建立基础图像读取与保存能力。
- 支持基础滤镜处理。
- 支持亮度、对比度、清晰度等基础调整。
- 建立早期单脚本处理流程。

### 商业价值

- 让普通 AI 图片具备初步可用性。
- 为 PPT 设计、基础视觉素材整理提供最早期的自动化能力。
- 验证本地化图像处理流程的可行性。

### 技术升级

- 使用 Python 与 OpenCV 建立基础处理链。
- 建立早期 `scripts/master_process.py` 脚本入口。
- 形成输入路径、输出路径、图像编码的基础逻辑。

### 输出能力

- 支持单图处理。
- 支持基础图片输出。
- 输出质量以“可用”为主，尚未形成商业高级感标准。

### 当前局限

- 视觉逻辑偏传统滤镜。
- 无法区分不同商业视觉类型。
- 缺少 AI 脏感识别。
- 缺少材质、光影和品牌风格判断。

### 下一版本方向

- 建立更稳定的锐化、降噪、色彩优化能力。
- 提升输出清晰度与基础质感。

## V2.0：高级增强阶段

### 版本定位

V2.0 从基础滤镜升级为多步骤图像增强流程，重点解决清晰度、噪点、色彩和基础 HDR 表现。

### 核心升级

- 高级锐化。
- 降噪系统。
- 色彩优化。
- HDR 增强。
- 初步高分辨率输出方向。

### 商业价值

- 让 AI 图片从“能看”提升到“较清晰、较干净”。
- 适合基础 PPT 封面、演示视觉、简单商业素材增强。
- 为后续商业视觉引擎化奠定基础。

### 技术升级

- 将处理流程拆分为更明确的算法步骤。
- 引入高频控制、微对比增强、色彩调整等方向。
- 开始保留历史脚本与兼容入口。

### 输出能力

- 支持更高分辨率输出。
- 输出图像更清晰。
- 对噪声与色彩有初步控制能力。

### 当前局限

- 仍然偏“图片增强工具”。
- 容易将 AI 假纹理锐化为脏细节。
- 缺少商业摄影光感。
- 缺少材质分类处理能力。

### 下一版本方向

- 从单脚本增强升级为工程化架构。
- 引入 mode、rules、analysis、quality_report 等系统。

## V3.0：AI 商业视觉引擎阶段

### 版本定位

V3.0 是 VisualMasterPro 从增强脚本升级为 AI 商业视觉质量引擎的关键版本，开始建立可扩展的软件架构。

### 核心升级

- AI 视觉增强。
- 商业质感优化。
- 批量处理框架。
- `main.py` 统一入口。
- `engine.pipeline` 主编排。
- 内部质量报告系统。

### 商业价值

- 项目定位从“清晰化工具”升级为“商业视觉质量引擎”。
- 开始支持不同商业视觉类型的处理方向。
- 为设计工作室、AI 视觉创作者和商业设计团队建立工程基础。

### 技术升级

- 新增 `engine/analysis/`、`engine/algorithms/`、`engine/pipeline/`。
- 建立视觉分析指标：AI 脏感、高频污染、中频结构、高光风险、色彩统一、空气感。
- 将旧脚本能力迁移到可扩展管线。

### 输出能力

- 支持 mode 参数。
- 支持质量报告。
- 支持中文路径。
- 支持后续 EXE 打包。

### 当前局限

- 各 mode 之间视觉差异仍不明显。
- 质量评分仍是启发式指标。
- 商业光影与材质重建仍处于早期。

### 下一版本方向

- 引入视觉规则驱动。
- 建立独立 mode pipeline。
- 收敛正式商业交付输出。

## V3.5：文件管理与商业流程稳定阶段

### 版本定位

V3.5 聚焦项目文件管理、输出结构规范和商业交付流程稳定，让项目从实验状态进入长期可维护状态。

### 核心升级

- 高端品牌视觉风格方向。
- 光影氛围系统。
- 材质增强方向。
- 文件管理规范。
- 输出目录分层。

### 商业价值

- 项目更适合 GitHub 管理、版本记录和团队协作。
- 正式输出、测试输出、调试输出不再混在一起。
- 更符合商业软件长期维护方式。

### 技术升级

- 建立 `pipelines/` 模式管线层。
- 新增 `docs/file_management_rules.md`。
- 输出结构开始区分正式输出、测试输出、调试输出。

### 输出能力

- 正式输出路径更清晰。
- 测试结果进入 `tests/outputs/`。
- 支持调试报告与正式交付分离。

### 当前局限

- 默认交付仍需进一步收敛。
- 用户端交付体验尚未完全稳定。

### 下一版本方向

- 建立单图商业交付系统。
- 默认只输出最终成品图。
- 将质量报告转为内部调试系统。

## V3.6：单图商业交付阶段

### 版本定位

V3.6 建立 Single Delivery Output System，使 VisualMasterPro 默认输出更符合商业软件交付逻辑。

### 核心升级

- 默认只输出一张最终商业交付图。
- JSON、Markdown、compare 图仅在 `--debug` 或 `--developer` 模式输出。
- 最终图右下角自动生成商业信息角标。
- 输出命名统一为 `原文件名_mode_雪原Ai·PPT设计.png`。

### 商业价值

- 从工程测试输出升级为商业最终交付输出。
- 减少用户看到的无关工程文件。
- 更适合客户预览、视觉交付和商业项目存档。

### 技术升级

- 新增 `engine/delivery/`。
- 新增商业信息角标模块。
- 新增开发调试对比图模块。
- `quality_report` 默认变为内部系统。

### 输出能力

- 正式输出进入 `输出成品/images/`。
- 默认只输出最终图。
- 开发模式可输出 JSON、Markdown 与 compare 图。

### 当前局限

- 角标字体仍可继续提升。
- 默认单图交付不适合批量任务，后续需要显式 `--batch`。
- 材质重建状态仍是交付提示，不是独立材质重建开关。

### 下一版本方向

- 建立商业签名引擎。
- 增强不同 mode 的视觉差异。
- 引入品牌视觉预设与商业级参数控制。

## V3.7：商业签名引擎阶段

### 版本定位

V3.7 Commercial Signature Engine 是面向商业化的关键版本，目标是让每个视觉模式形成可识别、可复用、可规模化的商业视觉签名。

### 核心升级

- 商业签名引擎。
- 品牌视觉预设。
- 高端商业风格统一系统。
- 商业级视觉参数控制。
- Premium Presets。
- 批量商业化处理支持。

### 商业价值

- 形成可收费的高级风格系统。
- 支持品牌 KV、商业海报、产品图、人像图、PPT 封面等批量标准化处理。
- 为创作者版、工作室版、企业版建立产品能力边界。

### 技术升级

- 建立 signature profile 结构。
- 将 mode pipeline 与 preset system 连接。
- 为不同行业建立可复用视觉签名。
- 为后续 GPU 管线和实时预览预留结构。

### 输出能力

- 支持更明确的品牌化输出。
- 支持商业级预设控制。
- 支持批量商业化处理方向。

### 当前局限

- 需要高质量行业参考图集。
- 需要更稳定的材质识别。
- 需要建立视觉回归测试标准。

### 下一版本方向

- V3.8 AI 场景智能。
- 引入场景理解、构图判断和自动光影平衡。
